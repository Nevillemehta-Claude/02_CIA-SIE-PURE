  • OPERATOR WORKFLOW: Copy login URL → Paste in Chrome → Login → Copy callback URL → Paste back

UXMI Golden Rules (CR-005)

  • Rule 1: Every interactive element has a tooltip describing its purpose
  • Rule 2: Every action has visible feedback within 100ms
  • Rule 3: Every wait >100ms shows a progress indicator
  • Rule 4: Every error explains WHAT happened, WHY, and HOW TO FIX
  • Rule 5: Every disabled element explains WHY via tooltip
  • Rule 6: Every function has keyboard access (Tab, Enter, Escape)
  • Rule 7: Every animation is smooth, purposeful, and respects motion preferences
  • REFERENCE: Full spec in ADDENDUM_002_UX_MICRO_INTERACTIONS.md

System State Machine

Kite Token Lifecycle

State Transition Rules

  • System: Only READY → IGNITION when ALL 12 pre-flight checks pass (green status)
  • System: IGNITION duration: exactly 3 seconds with countdown display
  • Kite Token: WARNING triggers yellow pulse animation in UI
  • Kite Token: CRITICAL triggers red alert banner + audio warning
  • Kite Token: EXPIRED blocks all operations, forces SHUTDOWN state

Implementation Guide for Claude Code

Step-by-step instructions to initialize and build the MCI project.

Phase 5: Implementation Sequence

1
Project Initialization

Open Claude Code CLI and execute the following commands:

mkdir mci && cd mci bun init -y bun add hono @hono/node-server bun add react react-dom bun add -d vite @vitejs/plugin-react tailwindcss postcss autoprefixer typescript @types/react @types/react-dom
2
Create Directory Structure

Scaffold the project according to the C3 Component Diagram:

src/ ├── server/ │ ├── index.ts # Hono entry (port 8080) │ ├── routes/ │ │ ├── health.ts # /api/health/* │ │ ├── ignition.ts # /api/ignition/* │ │ └── telemetry.ts # /api/telemetry/stream (SSE) │ └── services/ │ ├── scanner.ts # 12 pre-flight checks │ ├── kite.ts # Token lifecycle │ └── anthropic.ts # Model selection ├── client/ │ ├── App.tsx │ ├── components/ │ └── hooks/ └── shared/ └── types.ts # Shared TypeScript interfaces
3
Build Pre-Ignition Scanner First

Implement the 12 checks in scanner.ts. Each check must return:

interface CheckResult { id: string; // e.g., "PORT_8000" name: string; // Human-readable name status: 'pass' | 'fail' | 'warn'; latency_ms: number; // Execution time details: string; // Additional context } // All 12 checks must complete in < 500ms total async function runAllChecks(): Promise
4
Implement State Machine

Create the system state machine following the state diagram:

type SystemState = | 'DORMANT' | 'PRE_FLIGHT' | 'READY' | 'IGNITION' | 'ACTIVE' | 'SHUTDOWN'; // Transitions are strictly controlled // READY → IGNITION only when ALL checks pass
5
Build React Dashboard

Create the aerospace-themed control panel with:

Components Required: ├── PreIgnitionScanner.tsx # 12-check grid with real-time status ├── ModelSelector.tsx # Dropdown with capability tooltips ├── IgnitionButton.tsx # Gated by scanner + pulse animation ├── TelemetryDashboard.tsx # SSE-powered live metrics └── KiteTokenCountdown.tsx # HH:MM:SS with color transitions Design Tokens: - Background: #0a0e17 (Deep space black) - Accent: #06b6d4 (Cyan glow) - Success: #10b981 (Green) - Warning: #f59e0b (Amber) - Error: #ef4444 (Red)
6
Constitutional Constraints Enforcement

Create middleware that enforces inviolable rules:

// CR-001: ZERO trading signal access // Block any endpoint that could expose trading data // CR-002: Expose contradictions // Log and display any conflicting states without auto-resolution // CR-003: Descriptive only // All outputs must be READ-ONLY status displays // CR-004: Token accuracy // Hard-code 6:00 AM IST as expiry reference const KITE_EXPIRY_HOUR_IST = 6;
